# Агент анализа политических новостей Tengrinews.kz

## 1. Установка

1. Установите Python 3.10+ (если ещё не установлен): https://www.python.org/downloads/
2. Скачайте папку `tengrinews_agent` целиком на компьютер.
3. Откройте терминал (cmd/PowerShell на Windows, Terminal на macOS/Linux) в этой папке и выполните:

```bash
pip install -r requirements.txt
```

## 2. Ключ Google Gemini API (бесплатно)

Скрипт использует Gemini API для анализа новостей — это бесплатно в пределах
щедрого дневного лимита (более чем достаточно для одного отчёта в день).

1. Зайдите на https://aistudio.google.com/apikey (нужен обычный Google-аккаунт).
2. Нажмите "Create API key" — ключ создастся мгновенно, без привязки карты.
3. Задайте его как переменную окружения:

**Windows (PowerShell):**
```powershell
setx GEMINI_API_KEY "ваш-ключ-сюда"
```
(после этого перезапустите терминал)

**macOS / Linux (bash/zsh) — добавьте в `~/.bashrc` или `~/.zshrc`:**
```bash
export GEMINI_API_KEY="ваш-ключ-сюда"
```
и выполните `source ~/.bashrc`.

Бесплатный тариф Gemini 2.5 Flash ограничивает число запросов в минуту и в день,
но для одного запуска в сутки этого более чем достаточно. Если лимит вдруг
будет исчерпан (маловероятно при таком объёме), скрипт покажет ошибку с кодом
429 — в этом случае можно просто подождать сутки, лимит на бесплатном тарифе
обновляется ежедневно.

## 3. Ручной запуск (для проверки)

```bash
python tengrinews_analyzer.py
```

Если всё работает, в папке `reports/` появится файл вида:
`tengrinews_politics_report_2026-07-14.docx`

## 4. Автозапуск каждое утро в 10:00 (по Астане) через GitHub Actions

Этот вариант работает полностью в облаке GitHub — ваш компьютер даже не
обязан быть включён. Расписание уже настроено в файле
`.github/workflows/daily-analysis.yml` на `05:00 UTC`, что соответствует
`10:00` по времени Астаны (Казахстан не переводит часы, смещение +5 постоянно).

### Шаг 1. Создайте репозиторий на GitHub

1. Зайдите на https://github.com и создайте аккаунт, если его ещё нет.
2. Нажмите "New repository" → назовите, например `tengrinews-agent` →
   можно сделать приватным (Private) — это разумно, раз внутри будет
   логика с вашим ключом.

### Шаг 2. Загрузите файлы агента в репозиторий

Проще всего через сайт GitHub: "Add file" → "Upload files" → перетащите
всю папку `tengrinews_agent` (включая скрытую папку `.github`) → Commit.

Либо через git в терминале:
```bash
cd tengrinews_agent
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/tengrinews-agent.git
git push -u origin main
```

### Шаг 3. Добавьте API-ключ как секрет репозитория

1. В репозитории откройте: Settings → Secrets and variables → Actions.
2. Нажмите "New repository secret".
3. Имя: `GEMINI_API_KEY`, значение — ваш ключ с https://aistudio.google.com/apikey
4. Сохраните.

(Секреты в GitHub Actions зашифрованы и не видны в логах — это безопасное место
для ключа, в отличие от хранения его прямо в коде.)

### Шаг 4. Проверьте вручную

1. Вкладка "Actions" в репозитории → выберите workflow "Daily Tengrinews
   Political Analysis" → кнопка "Run workflow" (доступна благодаря
   строчке `workflow_dispatch` в файле workflow) → запустить.
2. Через минуту-две в разделе "Actions" появится лог выполнения. Если всё
   прошло успешно — в папке `reports/` вашего репозитория появится новый
   `.docx` файл, и он будет закоммичен автоматически.

### Шаг 5. Дальше — ничего не делать

С этого момента GitHub сам будет запускать анализ каждое утро в 10:00 по
Астане, коммитить новый отчёт в папку `reports/` и обновлять `last_run.json`,
чтобы не анализировать одни и те же новости дважды. Вам останется только
иногда заходить в репозиторий и скачивать новые отчёты (или настроить
уведомления на email о новых коммитах — GitHub делает это через
Watch → Custom → Pushes).

**Важно:** GitHub Actions автоматически отключает scheduled-задачи, если
в репозитории не было ни одной активности (коммита) 60 дней подряд. Раз
скрипт сам коммитит отчёты, он будет "продлевать себе жизнь" каждый день,
пока находит новости — но если новостей долго не будет, стоит иногда
заходить и делать любой коммит вручную.

## 5. Как это работает по сути

- Скрипт помнит дату последнего запуска (файл `last_run.json`, который
  сам коммитится обратно в репозиторий), поэтому каждое утро анализирует
  именно новые статьи, вышедшие со вчерашнего дня, а не всё подряд.
- Каждый запуск с новыми статьями создаёт отдельный docx-файл в `reports/` —
  у вас будет накапливаться архив ежедневных отчётов с историей.
- Если за сутки не было ни одной новой политической статьи — отчёт не
  создаётся, скрипт просто обновляет дату последней проверки.
- Никакого участия человека после настройки не требуется.

## Альтернатива: локальный запуск на своём компьютере

Тот же скрипт можно вместо GitHub Actions запускать локально через
Планировщик заданий Windows или cron на macOS/Linux — просто ставите
`python tengrinews_analyzer.py` на нужное расписание. Скажите, если такой
вариант тоже понадобится — дам инструкцию.
